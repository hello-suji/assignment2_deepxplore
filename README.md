# Assignment 2 - DeepXplore 스타일 차등 테스트

## 1. 개요

본 프로젝트는 DeepXplore의 아이디어를 기반으로 딥러닝 모델에 대한 차등 테스트(differential testing)를 수행한다.  
CIFAR-10 데이터셋으로 학습된 두 개의 ResNet50 모델을 사용하여 서로 다른 예측을 수행하는 입력을 탐지하고, 뉴런 커버리지(neuron coverage)를 측정한다.

본 실험의 목적은 동일한 입력에 대해 서로 다른 예측을 생성하는 경우를 탐지함으로써, 모델의 오류 가능성이 있는 입력이나 경계 사례를 식별하는 것이다.

## 4. 모델 구성

* 모델 구조: ResNet50
* 데이터셋: CIFAR-10
* 모델 개수: 2개

각 모델은 서로 다른 설정으로 독립적으로 학습되었다.

| 모델      | 랜덤 시드 | 학습률    |
| ------- | ----- | ------ |
| Model A | 42    | 0.001  |
| Model B | 123   | 0.0005 |

또한 CIFAR-10에 맞게 ResNet50의 마지막 fully connected layer를
1000 클래스에서 10 클래스로 수정하였다.

---

## 5. 구현 세부 사항 (DeepXplore 적용)

본 구현은 DeepXplore의 핵심 개념을 다음과 같이 적용하였다.

### 5.1 차등 테스트

* 동일한 입력을 두 모델에 동시에 입력
* 서로 다른 예측을 생성하는 입력을 수집
* 이러한 입력을 오류 가능성이 있는 사례로 간주

---

### 5.2 뉴런 커버리지

* ReLU 활성값을 기반으로 커버리지 측정
* 특정 임계값 이상으로 활성화된 뉴런을 “커버됨”으로 간주
* 커버리지는 다음과 같이 계산된다

```
(활성화된 뉴런 수) / (전체 뉴런 수)
```

* 단순한 방식이므로 커버리지는 빠르게 1.0에 도달할 수 있음

---

### 5.3 주요 수정 사항

* CIFAR-10 이미지를 ResNet50 입력 크기에 맞게 리사이즈
* 출력 레이어를 1000 → 10 클래스로 수정
* forward hook을 이용한 뉴런 커버리지 직접 구현
* ResNet 구조에서 발생하는 activation shape 문제 해결 (shape 기반 key 분리)
* DeepXplore 원본 코드 대신 PyTorch 기반으로 직접 구현

---

### 5.4 수정이 필요한 이유

* 기존 DeepXplore 구현은 오래된 라이브러리에 의존하여 최신 PyTorch 환경과 호환되지 않음
* CIFAR-10과 ResNet50 구조는 원본 DeepXplore의 입력 설정과 다름
* 따라서 동일한 핵심 아이디어(차등 테스트 + 뉴런 커버리지)를 유지하면서 PyTorch 기반으로 재구현함

## 7. 결과 분석

### 7.1 관찰된 특징

차등 테스트 수행 결과, 여러 개의 disagreement-inducing input이 발견되었다.
이들 입력은 다음과 같은 특징을 보였다.

* 시각적으로 모호한 이미지에서 불일치가 자주 발생
* 서로 유사한 클래스 간 혼동 (예: cat vs dog, automobile vs truck)
* 객체 경계가 불명확하거나 해상도가 낮은 이미지

---

### 7.2 불일치 발생 원인

다음과 같은 이유로 모델 간 예측 차이가 발생한 것으로 판단된다.

1. **결정 경계 근처 입력 (Decision boundary)**

   * 두 모델이 서로 다른 경계 기준을 학습하여 예측이 갈림

2. **학습 초기화 및 하이퍼파라미터 차이**

   * 서로 다른 seed와 learning rate로 인해 feature representation이 달라짐

3. **낮은 학습 안정성**

   * epoch 수가 적어 모델이 충분히 수렴하지 않음

---

### 7.3 뉴런 커버리지 해석

두 모델 모두 뉴런 커버리지가 1.0에 도달하였다.

이는 다음을 의미한다.

* 기본 neuron coverage metric은 매우 빠르게 포화됨
* 높은 coverage가 반드시 테스트 품질이 높음을 의미하지 않음
* 더 정교한 coverage metric (예: k-multisection coverage)이 필요할 수 있음

---

### 7.4 요약

차등 테스트는 모델 간 예측 불일치를 효과적으로 발견하였으며,
이를 통해 모델의 취약 입력을 식별할 수 있었다.

이는 DeepXplore 방식이 모델 테스트에 유용하게 활용될 수 있음을 보여준다.
